Um novo endpoint Showcase foi criado para fornecer uma interface pública (GET)
para clientes frontend/mobile consultarem produtos de forma otimizada.
CARACTERÍSTICAS PRINCIPAIS:
✅ Listagem com paginação obrigatória
✅ Filtros opcionais (busca, brand, categoria)
✅ Detalhes completos do produto
✅ Isolamento por tenant (multi-tenancy)
✅ Preços otimizados (mínimo e promocional)
✅ Endpoints somente leitura (GET)
✅ Validação rigorosa de entrada
✅ Logging estruturado
✅ Sem autenticação (dados públicos)

                              ENDPOINTS IMPLEMENTADOS
═══════════════════════════════════════════════════════════════════════════════
1. LISTAR PRODUTOS COM PAGINAÇÃO E FILTROS
   GET /api/v1/showcase/products
   Headers:
   - X-Tenant-ID: {UUID} [OBRIGATÓRIO]
   - Content-Type: application/json
   Query Parameters:
   - page: int (default: 0, zero-based)
   - size: int (default: 20, max: 100)
   - search: string (opcional, busca em name e description)
   - brandId: UUID (opcional)
   - categoryId: UUID (opcional)
   Response: 200 OK
   {
     "content": [
       {
         "id": "...",
         "name": "...",
         "description": "...",
         "slug": "...",
         "brand": { "id": "...", "name": "..." },
         "category": { "id": "...", "name": "..." },
         "tags": ["tag1", "tag2"],
         "price": {
           "basePrice": null,
           "lowestVariantPrice": { "amount": 1999.99, "currency": "BRL" },
           "lowestPromotionalPrice": { "amount": 1499.99, "currency": "BRL" }
         },
         "variants": [...]
       }
     ],
     "pageInfo": {
       "page": 0,
       "size": 20,
       "totalPages": 5,
       "totalElements": 95
     }
   }
2. OBTER DETALHES DO PRODUTO
   GET /api/v1/showcase/products/{id}
   Path Parameters:
   - id: UUID [OBRIGATÓRIO]
   Response: 200 OK
   {
     "id": "...",
     "name": "...",
     "description": "...",
     "variants": [
       {
         "id": "...",
         "skuCode": "...",
         "barcode": "...",
         "price": { "amount": ..., "currency": "..." },
         "promotionalPrice": { "amount": ..., "currency": "..." },
         "dimensions": { "weight": ..., "height": ..., "width": ..., "depth": ... }
       }
     ]
   }

                            COMO USAR O ENDPOINT
═══════════════════════════════════════════════════════════════════════════════
1. COMPILAR:
   cd mscatalog
   mvn clean compile -DskipTests
   mvn clean package -DskipTests
2. EXECUTAR:
   ./mvnw spring-boot:run
   # ou
   java -jar target/mscatalog-0.0.1-SNAPSHOT.jar
3. TESTAR:
   # Listar produtos
   curl -X GET "http://localhost:8080/api/v1/showcase/products?page=0&size=20" \
     -H "X-Tenant-ID: f81d4fae-7dec-11d0-a765-00a0c91e6bf6"
   # Buscar por termo
   curl -X GET "http://localhost:8080/api/v1/showcase/products?search=notebook" \
     -H "X-Tenant-ID: f81d4fae-7dec-11d0-a765-00a0c91e6bf6"
   # Detalhes do produto
   curl -X GET "http://localhost:8080/api/v1/showcase/products/123e4567-e89b-12d3-a456-426614174000"
   # Script completo de testes
   chmod +x test-showcase.sh
   ./test-showcase.sh
